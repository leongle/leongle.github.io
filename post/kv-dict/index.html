主要记录了UPMesh中的关键键值对及取值解释。

应答码

100以内应答码由服务网格定义使用，100到255之间应答码由应用可选使用，服务网格目前定义的应答码如下：

应答码HEX 应答码DEC 含义
0x00 00 成功
0x30 48 无可用服务节点
0x31 49 通讯链路不可用（app<–>mproxy）
0x32 50 无调用权限
0x33 51 指定路由非法
0x34 52 SDK调用参数错误
0x40 64 未订阅指定服务
0x41 65 协议头格式错误
0x50 80 系统异常（无法归类时提供）
0x51 81 服务停用（兼容magpie）
0x52 82 调用超时
0x53 83 流量控制

LOGID生成规则

UPMesh各模块logid生成规则 ：前缀(3位子系统代码+1位保留)+时间戳mmddHHMMSS（10位）+序号（8位）

logid前缀 模块
USM0 console
USM0 pilot(ROOT CU)
USM0 pilot(CU)
USM0 mixer
USM0 monitor
USM0 magent
USM0 mproxy
USM0 meshctl

UMP告警

watch_dogs接口

UPMesh各模块目前通过mixer的watch_dogs接口和UMP系统进行对接，各字段取值说明如下：

字段 字段含义 格式或说明
cmd 命令字 固定填写 watch_dogs
role 角色 复合结构，所有和mixer交互的接口都会有这个字段
level 告警级别 通常有FATAL/ERROR级别，WARN/INFO/DEBUG通常不出现
system 系统代码 向Zdogs的申请的系统代码，不超过5个字，UPMesh为 UPSM
subsystem 子系统码 自定义的子系统代码，不超过5个字，建议的子系统 PILOT/MIXER/MON/CON/SC
module 模块代码 自定义的模块代码，不超过3个字节，建议的模块代码 PLT/MIX/MON/CON/AGT/PRX
code 事件码 自定义的事件码，不超过4个字节，取值见3.2节说明
msg 告警内容 UMP告警内容，参见3.3节说明
fileName 文件名 告警来自哪个代码文件， 不超过25个字节
function 函数名 告警来自哪个函数， 不超过24个字节
lineNo 代码行号 告警来自代码文件的哪一行， 不超过6个字节

事件码规则

事件码总长4个字节，采用1字节大类+1字节小类+2字节序号的编码规则。

大类用于区分不同的应用模块，分类规则如下：

大类 模块
C console
P pilot
M mixer
A magent
X mproxy
N monitor

小类用于区不同的事件对象，规则如下：

小类 事件对象
1 数据库相关
2 Redis相关
3 网络相关
4 其他未归类
